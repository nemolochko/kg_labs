# Лабораторная работа №1 Разработка приложения для конвертации между цветовыми моделями

**Цель работы:** Создание графического приложения на Java для конвертации цветов между моделями RGB, HSL и CMYK с использованием библиотеки Swing.

## Описание программы

Программа `App` представляет собой графическое приложение, реализующее преобразование цветов между тремя цветовыми моделями: RGB (Красный, Зеленый, Синий), HSL (Тон, Насыщенность, Яркость) и CMYK (Голубой, Пурпурный, Желтый, Черный). Интерфейс приложения предоставляет интуитивно понятный способ просмотра и редактирования цветов с использованием слайдеров и текстовых полей ввода.

## Функциональность

**Конвертация между цветовыми моделями:**
- RGB - компоненты R, G, B в диапазоне 0-255
- HSL - компоненты H (0-360°), S (0-100%), L (0-100%)
- CMYK - компоненты C, M, Y, K в диапазоне 0-100%

**Элементы управления:**
- Три панели со слайдерами и текстовыми полями для каждой цветовой модели
- Визуальная панель предпросмотра текущего цвета
- Синхронизированное отображение значений во всех моделях

**Особенности реализации:**
- Автоматическое обновление всех компонентов при изменении любой цветовой модели
- Защита от циклических обновлений через флаг `isUpdating`
- Валидация вводимых в текстовые поля значений
- Округление значений для целочисленного представления

## Техническая реализация

**Интерфейс:**
- Главное окно наследуется от `JFrame`
- Используется `BorderLayout` для компоновки элементов
- Панели цветовых моделей используют `GridLayout`
- Для слайдеров применяются `ChangeListener`, для текстовых полей - `ActionListener`

**Алгоритмы преобразования:**

**RGB → CMYK:**
```java
double k = 1 - Math.max(rd, Math.max(gd, bd));
double c = (1 - rd - k) / (1 - k + 1e-10);
double m = (1 - gd - k) / (1 - k + 1e-10);
double y = (1 - bd - k) / (1 - k + 1e-10);
```

**CMYK → RGB:**
```java
int r = (int)Math.round(255 * (1 - c) * (1 - k));
int g = (int)Math.round(255 * (1 - m) * (1 - k));
int b = (int)Math.round(255 * (1 - y) * (1 - k));
```

**RGB → HSL:**
```java
double max = Math.max(rd, Math.max(gd, bd));
double min = Math.min(rd, Math.min(gd, bd));
double l = (max + min) / 2.0;
double d = max - min;
s = d / (1 - Math.abs(2 * l - 1));
```

**HSL → RGB:**
```java
double c = (1 - Math.abs(2 * l - 1)) * s;
double x = c * (1 - Math.abs((h / 60) % 2 - 1));
double m = l - c / 2;
```

## Использование приложения

1. **Изменение в RGB:** Перемещение слайдеров R, G, B или ввод значений в текстовые поля
2. **Изменение в HSL:** Регулировка тона (H), насыщенности (S) и яркости (L)
3. **Изменение в CMYK:** Настройка компонент голубого, пурпурного, желтого и черного цветов
4. **Визуализация:** Текущий цвет отображается на центральной панели предпросмотра

Все изменения автоматически синхронизируются между моделями, обеспечивая согласованное представление цвета в разных системах.

## Структура программы

- **Основной класс `App`** - наследуется от JFrame и содержит всю логику приложения
- **Методы создания UI:** `createRGBHSLPanel()`, `createCMYKPanel()`, `createSlider()`, `createTextField()`
- **Методы связывания:** `bindSliderToField()` - обеспечивает синхронизацию слайдеров и текстовых полей
- **Методы конвертации:** `rgbToCmyk()`, `cmykToRgb()`, `rgbToHsl()`, `hslToRgb()`
- **Метод обновления:** `updateFromRGB()` - централизованное обновление всех компонентов
